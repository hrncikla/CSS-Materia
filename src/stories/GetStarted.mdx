# Get Started

You can use My Materia in two main ways:

### a) Include raw CSS (quickest)

Simply copy the built CSS into your project and include it in your HTML:

```html
<link rel="stylesheet" href="/css/materia.min.css" />
```

### b) Use SCSS (recommended for customization)

You have two main options, depending on how much control you want over your project structure:

**Option 1: Full Material Folder**  
For full integration with Storybook and all components, clone or copy the entire `materia/` folder:

```bash
CSS-Materia/
├── .storybook/      
├── build/           
├── public/                    
├── src/
│  └── ...
│  └── scss/
│     └── main.scss
├── ...
```

**Option 2: Just SCSS Folder**  
If you only need the core styles, you can copy just the `scss/` directory:

```bash
src/
 ├── scss/
 │    ├── tokens/
 │    ├── mixins/
 │    ├── layout/
 │    ├── components/
 │    └── base/
 │    └── main.scss
```

Then import it in your main SCSS file:

```scss
@use "materia/base";
@use "materia/tokens" as tokens;
@use "materia/components/button";
```

You can override tokens or mixins before importing components.

## 3. Customize Tokens

To adapt Materia to your brand, edit file `src/scss/tokens/user-tokens.scss`.  
It includes colors, typography, radius, spacing, etc.

## 4. Build & Storybook

Before you can start Storybook, you **must** build your CSS. Use the following scripts from your `package.json`:

```json
"scripts": {
  "storybook": "npm run build:css && storybook dev -p 6006",
  "build-storybook": "storybook build",
  "build:css": "sass src/scss/main.scss build/css/materia.min.css --style compressed --no-source-map",
  "watch:css": "sass --watch src/scss:build/css"
}
```

### Important:
- **Build CSS first**: Run `npm run build:css` before starting Storybook.
- **Rebuild after changes**: Run `npm run build:css` again whenever you update your SCSS files to reflect changes in Storybook.
